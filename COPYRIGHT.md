# Copyright — 뉴스 저작권 정책

요잇슈 일일 뉴스 파이프라인은 네이버 뉴스 검색 API 로 수집한 기사 URL 에 대해 본문을 크롤링·요약하여 캐릭터 대화를 생성한다. 원저작자의 저작권을 존중하기 위한 최소 방어선을 이 문서에 기록한다.

**근본 원칙**: 저작권 분쟁이 발생하면 콘텐츠 사업의 존폐가 걸리므로, **회색지대는 차단**한다. 유료·구독 기반 매체는 무단 본문 크롤링·요약이 저작권법 위반 + 약관 위반이 되므로 파이프라인 진입 자체를 막는다.

**정책 전환 (2026-04-22)**: 블랙리스트 → **화이트리스트**. 새 유료화 매체를 끊임없이 추적해야 하는 블랙리스트 구조의 누락 위험이 한경 사고로 드러남. 이후부터는 **검증된 안전 매체만 허용**.

## 차단 문구 (process-news.js `isInvalidContent` INVALID_PATTERNS)

화이트리스트 매체라도 일부 기사가 유료·프리미엄 섹션일 수 있으므로 본문 head 300자에서 페이월/프리미엄 문구가 발견되면 해당 기사 skip. 2차 방어선.

**일반 페이월**
- `유료\s*(기사|콘텐츠|구독)`
- `구독(자|회원).*이용`
- `로그인\s*(후|하면|하고)\s*(이용|열람|확인|보실)`
- `회원\s*가입\s*(후|하면|하고)`

**프리미엄 랜딩**
- `프리미엄\s*콘텐츠`
- `독자적인\s*(프리미엄|콘텐츠)`
- `구독자?\s*전용`
- `멤버십\s*전용`
- `결제(가)?\s*(필요|후|하고)`
- `Premium\s*only` / `Subscribers?\s*only`

**매체 고유 (한경 등 과거 사고 기반)**
- `한경\s*AI`
- `회원이?\s*되어\s*보세요`
- `지금\s*바로\s*(가입|구독)`

## 운영 룰

1. **화이트리스트 확장은 신중히**: 새 매체를 추가하려면 ① 유료/프리미엄 섹션 없는지 ② 품질·편향이 주류 수준인지 ③ 대화형 앱 독자에게 적절한지 3가지를 확인.
2. **정기 검토 불필요 → 단, 기존 허용 매체가 유료화하면 즉시 제거**. 화이트리스트는 "안전한 것만 들어옴" 구조라 새 매체 추적 부담은 없음. 다만 `kbs.co.kr` 같은 허용 매체가 갑자기 유료화 한다면 목록에서 빼야 함.
3. **사건 기록**: 저작권 관련 문제가 발생하면 본 문서 "대응 이력" 표에 날짜·사건·조치 3열로 기록.
4. **이중 방어선**: 화이트리스트(1차) + 본문 패턴(2차). 도메인은 URL 만 보고 빠르게 컷, 본문 패턴은 화이트 내 유료 섹션 기사를 추가 컷.

## 재발 방지 체크리스트

- [ ] 본문 head 300자 이내에 저작권 의심 문구가 없는지 `isInvalidContent` 로 검증되는가
- [ ] 화이트리스트 매체가 유료화 / 페이월 도입을 했는지 주요 사건 발생 시 확인하는가
- [ ] 새 매체 추가 요청 시 위 "운영 룰 1" 체크리스트를 충족하는가
