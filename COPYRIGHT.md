# Copyright — 뉴스 저작권 정책

요잇슈 일일 뉴스 파이프라인은 네이버 뉴스 검색 API 로 수집한 기사 URL 에 대해 본문을 크롤링·요약하여 캐릭터 대화를 생성한다. 원저작자의 저작권을 존중하기 위한 최소 방어선을 이 문서에 기록한다.

**근본 원칙**: 저작권 분쟁이 발생하면 콘텐츠 사업의 존폐가 걸리므로, **회색지대는 차단**한다. 유료·구독 기반 매체는 무단 본문 크롤링·요약이 저작권법 위반 + 약관 위반이 되므로 파이프라인 진입 자체를 막는다.

## 차단 도메인 (select-news.js `BLOCKED_DOMAINS`)

유료·구독 페이월 매체 — 수집 단계에서 URL 기반 차단. `hostname.includes(d)` 로 판정하므로 서브도메인 자동 커버.

| 도메인 | 매체명 | 차단 사유 |
|--------|--------|-----------|
| hankyung.com | 한국경제 | 프리미엄 콘텐츠 유료화 |
| mk.co.kr | 매일경제 | 유료 구독 모델 |
| chosunbiz.com | 조선비즈 | 유료 구독 모델 |
| mt.co.kr | 머니투데이 | 유료 구독 모델 |
| sedaily.com | 서울경제 | 유료 구독 모델 |
| edaily.co.kr | 이데일리 | 유료 구독 모델 |

비 저작권 사유로 차단된 도메인(미디어오늘 등) 은 `select-news.js` 주석 참고.

## 차단 문구 (process-news.js `isInvalidContent` INVALID_PATTERNS)

본문 head 300자에서 매치되면 해당 기사 skip. BLOCKED_DOMAINS 를 우회해 들어온 경우에 대한 2차 방어선.

**일반 페이월**
- `유료\s*(기사|콘텐츠|구독)`
- `구독(자|회원).*이용`
- `로그인\s*(후|하면|하고)\s*(이용|열람|확인|보실)`
- `회원\s*가입\s*(후|하면|하고)`

**프리미엄 랜딩**
- `프리미엄\s*콘텐츠`
- `독자적인\s*(프리미엄|콘텐츠)`
- `구독자?\s*전용`
- `멤버십\s*전용`
- `결제(가)?\s*(필요|후|하고)`
- `Premium\s*only` / `Subscribers?\s*only`

**매체 고유 (한경)**
- `한경\s*AI`
- `회원이?\s*되어\s*보세요`
- `지금\s*바로\s*(가입|구독)`

## 운영 룰

1. **차단 우선**: 유료 여부가 애매하면 BLOCKED_DOMAINS 에 선등록 후 개별 검토.
2. **정기 확인**: 월 1회 `redline_logs` 의 통과 기사 URL 을 도메인별로 집계. 새 유료화 매체가 보이면 즉시 추가.
3. **사건 기록**: 저작권 관련 문제가 발생하면 본 문서 "대응 이력" 표에 날짜·사건·조치 3열로 기록.
4. **이중 방어선**: 도메인 차단 + 본문 패턴 차단. 도메인은 URL 만 보고 빠르게 컷, 본문 패턴은 느려도 확실하게 컷.

## 재발 방지 체크리스트

- [ ] 본문 head 300자 이내에 저작권 의심 문구가 없는지 `isInvalidContent` 로 검증되는가
- [ ] 통과된 기사의 hostname 이 BLOCKED_DOMAINS 에 없는지 재확인 루틴 있는가
- [ ] `redline_logs` 통과 섹션 출처 분포를 월 1회 검토하는가

## 연계 문서

- `REDLINE.md` — 콘텐츠 리스크(윤리·편향) 차단. 본 문서는 저작권 리스크 차단.
- `select-news.js` — Stage 1 수집·1차 필터.
- `process-news.js` — Stage 2 크롤링·2차 필터(`isInvalidContent`).
